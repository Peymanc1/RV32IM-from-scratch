RV32IM SoC — Çip-Üstü Datapath

Bu şema, işlemcinin çevresini (datapath'in "çip üstü" hâlini) gösterir: CPU'nun bellek portları → I-cache / D-cache → AXI burst master → SmartConnect → PS S_AXI_HP → DDR; ve veri portunun MMIO dalı → framebuffer / palet / LED / switch → HDMI. Oklardaki etiketler adresleri (DDR penceresi ve MMIO) gösterir. CPU çekirdeğinin iç datapath'i (PC, ALU, regfile, MUX'lar) için datapath.html.

CPU ↔ cache (komut/veri, bayt-adresli) AXI4 burst (satır ↔ DDR) MMIO (LED / framebuffer / palet / switch)

Adres haritası (decode kuralı)

rv32im_doom_core.sv içinde adresin üst hane'sine bakılır (is_ddr = (d_addr[31:28] == 4'h1)): 0x1… → DDR (cache→AXI→HP), aksi → mmio_bridge.

Akışın özeti

  • Komut çekme: CPU imem portu → I-cache. Hit → 1 cycle. Miss → burst master bir satır (32 B) DDR'dan çeker (cache→AXI→SmartConnect→S_AXI_HP→DDR controller→DDR).
  • Veri: CPU dmem portu → adres decode. 0x1… → D-cache (write-back) → DDR. 0x9/0xA/0xB → mmio_bridge.
  • Ekran: mmio_bridge'in framebuffer(0xA)/palet(0xB) yazmaları → video_fb (çift-saatli) → rgb2dvi (TMDS) → HDMI.
  • PS rolü: ARM kod çalıştırmaz; ps7_init sadece saatleri ve DDR controllersini açar. Program/WAD baytlarını DDR'a XSCT (dow, JTAG) yazar; CPU yalnızca okur.
